## 1. プロジェクトの目的と背景

### 1.1 ブロックチェーンに対する量子脅威

現在のブロックチェーン暗号プリミティブ（ECDSA、secp256k1）は、十分に強力な量子コンピュータ上で動作するショアのアルゴリズムに対して脆弱です。保守的な見積もりでは、256ビット楕円曲線を破る能力を持つ量子コンピュータは10〜15年以内に出現する可能性があり、以下を脅かします：

- **2兆ドル以上**の暗号資産
- すべての主要チェーンにおけるスマートコントラクトのセキュリティ
- クロスチェーンブリッジの完全性
- デジタル署名インフラストラクチャ

### 1.2 PQC + ZK 融合の必要性

解決策には、2つの暗号パラダイムの融合が必要です：

| 技術 | 役割 | 課題 |
|------|------|------|
| **PQC (Dilithium)** | 耐量子署名 | 大きな署名サイズ（約2.4KB）、高価なオンチェーン検証 |
| **ZK証明** | 簡潔な検証 | Dilithium演算の証明は計算負荷が高い |

**私たちのイノベーション:** 各証明システムの長所を活かした2段階証明パイプライン：
- **Plonky2**: バッチ集約のための高速・再帰的STARK
- **SP1**: L1効率的な検証のためのGroth16出力を持つzkVM

### 1.3 市場ポジショニング

以下を組み合わせた既存プロジェクトは存在しません：
1. NIST標準化されたPQC（FIPS 204 ML-DSA/Dilithium）
2. 暗号回路の形式検証
3. 2段階再帰証明アーキテクチャ
4. ガス最適化されたL1検証（<300Kガス）

これは、耐量子DeFiインフラストラクチャにおける真の先行者利益を表しています。

## 2. 技術的マイルストーン

### 2.1 フェーズ1: Dilithium NTT回路（完了）

**目標:** Dilithium署名検証をZKフレンドリーな算術回路で実装する。

| 指標 | 目標 | 達成 | ステータス |
|------|------|------|------------|
| NTT順変換/逆変換の正確性 | 100% | 100% | 合格 |
| Montgomery削減の境界 | ±q | 検証済み | 合格 |
| Barrett削減の境界 | [0, q) | 検証済み | 合格 |
| ネガティブテストカバレッジ | >15テスト | **19テスト** | 合格 |
| オーバーフロー検出 | 全パス | 全カバー | 合格 |

**主要な成果物:**
- `ntt.rs`: インプレースバタフライ演算を用いた順/逆NTT
- `reduce.rs`: 形式的境界を持つMontgomery（REDC）とBarrett削減
- `dilithium_circuit.rs`: 完全な検証回路

**検証ハイライト:**
```
19個のネガティブテストがカバー:
├── 無効な係数範囲
├── NTT長の不一致
├── 削減境界違反
├── 署名フォーマットエラー
└── 公開鍵検証失敗
```

## 4. Quantum Shield Bridgeの設計

### 4.1 コントラクトアーキテクチャ

```solidity
interface IQuantumVerifier {
    function verify(
        bytes calldata proof,
        bytes32 batchRoot,
        uint256 totalAmount,
        uint256 commitmentHash
    ) external view returns (bool);
}

contract QuantumShieldBridge {
    IQuantumVerifier public verifier;

    // 将来のPQCアルゴリズム更新のためのアップグレード可能な検証器
    function setVerifier(IQuantumVerifier _verifier) external onlyOwner;

    // 検証済みバッチの処理
    function processBatch(
        bytes calldata groth16Proof,
        bytes32 batchRoot,
        uint256 totalAmount,
        BridgeTransfer[] calldata transfers
    ) external;
}
```
