# 42: 統合スクリーンレビュー (Unified Screen Review)

> **目的**: 全画面を5つの観点から網羅的にレビューし、品質を保証する
> **実行者**: AI（Playwright MCP使用）
> **トリガー**: `画面レビュー 開始` - 進捗トラッカーから自動で続きを開始

---

## 概要

このプロンプトは、Quantum Shieldの全175画面を**5つの観点**から体系的にレビューするための統合ガイドです。

### 5つのレビュー観点

| # | 観点 | 責任者視点 | 主なチェック内容 |
|---|------|-----------|----------------|
| **D** | Design（デザイン） | CDO | ブランド一貫性、デザインシステム準拠、視覚品質 |
| **J** | Journey（ジャーニー） | PM/UX | ユーザーフロー、入口/出口、次アクションの明確さ |
| **N** | Navigation（遷移） | CTO | 全ボタン・リンクの動作、戻るボタン、ヘッダー/フッター |
| **M** | Model（データ整合） | CTO | DATA_MODEL.mdとの整合性、必要データの有無 |
| **C** | Completeness（完全性） | Product | あるべき機能、不要な要素、ツールチップ |

### D3: タップエリア44px以上

```javascript
// Playwright検証コード
() => {
  const elements = document.querySelectorAll('button, a, [role="button"], [tabindex="0"]');
  const issues = [];

  elements.forEach((el, i) => {
    const rect = el.getBoundingClientRect();
    if (rect.width > 0 && rect.height > 0) {
      if (rect.width < 44 || rect.height < 44) {
        issues.push({
          text: el.textContent?.substring(0, 30),
          width: Math.round(rect.width),
          height: Math.round(rect.height)
        });
      }
    }
  });

  return { total: elements.length, issues };
}
```

## STEP 3: J - ジャーニーレビュー（UX観点）

### ジャーニー位置の確認

**参照**: `docs/agents/prompts/01_journey_validator.md`

| ステップ | 該当画面例 | 確認内容 |
|---------|-----------|---------|

| Consideration | How It Works | 差別化ポイント理解できるか |
| Activation | Onboarding | 初回体験スムーズか |

| Main Task | Lock/Unlock | 操作が直感的か |
| Confirmation | Success | 成功を実感できるか |
| Retention | History | 継続利用の価値を感じるか |

### J1-J5: ジャーニーチェックリスト

```yaml
J1 - 入り口の存在:
  質問: この画面にどうやって来る？
  確認:
    - ナビゲーションメニューにリンクがある
    - 他画面のボタンから遷移できる
    - URL直接アクセス時の挙動が定義されている

J2 - 出口の存在:
  質問: この画面から次にどこへ行く？
  確認:
    - 主要CTAが存在する
    - 戻るボタンがある（詳細画面の場合）
    - ナビゲーションで他画面へ移動できる

J3 - 戻るボタンの一貫性:
  ルール:
    - フォームステップ → 前ステップ
    - 詳細画面 → 一覧画面
    - 確認画面 → 入力画面
    - 完了画面 → ダッシュボード

J4 - 次アクション明確:
  質問: ユーザーは次に何をすべきか分かるか？
  確認:
    - Primary CTAが目立つ
    - 説明テキストが十分
    - 迷わずに次に進める

J5 - デッドエンド検出:
  質問: 行き止まりになっていないか？
  確認:
    - エラー後に復帰手段がある
    - タイムアウト時のフォールバックがある
    - 「何もできない」状態がない
```

```yaml
  アクション: 次の画面へ進む

CONDITIONAL:
  条件: CONDITIONALが2-3つ、FAILなし
  アクション: 問題をIssue登録、次の画面へ進む

FAIL:
  条件: FAILが1つ以上
  アクション: 即時修正、再レビュー（PASSするまで繰り返し）
```

---

## FAIL時の修正フロー（必須）

**重要**: FAILの場合は次の画面に進まず、PASSするまで修正→再レビューを繰り返す。

```
┌─────────────────────────────────────────────────────────────────────┐
│  FAIL FIX FLOW                                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  STEP F1: 問題の優先順位付け                                         │
│  ├─ 深刻度「高」の問題を最優先                                      │
│  ├─ N（ナビゲーション）のエラーは最優先（ユーザー影響大）          │
│  └─ D（デザイン）のタップエリアは次点                              │
│                                                                     │
│  STEP F2: 修正実施                                                   │
│  ├─ 問題ごとに該当ファイルを特定                                   │
│  ├─ 修正を実施                                                      │
│  ├─ TypeScriptコンパイル確認                                       │
│  └─ 修正内容を記録                                                  │
│                                                                     │
│  STEP F3: 再レビュー                                                 │
│  ├─ 画面をリロード                                                  │
│  ├─ 修正した観点のみ再検証                                         │
│  └─ 結果を更新                                                      │
│                                                                     │
│  STEP F4: 判定更新                                                   │
│  ├─ 全てPASS/CONDITIONAL → トラッカー更新、次の画面へ              │
│  └─ まだFAILあり → STEP F1に戻る                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 修正優先順位

| 優先度 | 観点 | 問題タイプ | 理由 |
|:------:|------|-----------|------|
| 1 | N | 遷移先エラー（500, 404） | ユーザーが先に進めない |
| 2 | N | リンク切れ | ユーザーが迷う |
| 3 | D | タップエリア44px未満 | モバイルで操作困難 |
| 4 | C | 必須機能の欠如 | 機能不全 |
| 5 | M | データ不整合 | 表示エラーの原因 |
| 6 | J | デッドエンド | ユーザーが詰まる |

### 修正記録テンプレート

```markdown
## 修正記録: {画面名}

### 修正 #{N}
- **観点**: D / J / N / M / C
- **問題**: {問題の説明}
- **修正ファイル**: {ファイルパス}
- **修正内容**: {変更内容}
- **確認結果**: ✅ 解決 / ❌ 未解決

### 再レビュー結果
| 観点 | 修正前 | 修正後 |
|------|:------:|:------:|
| D | ❌ | ✅ |
| N | ❌ | ✅ |
| ... | ... | ... |

### 総合判定: PASS / CONDITIONAL / FAIL
```
