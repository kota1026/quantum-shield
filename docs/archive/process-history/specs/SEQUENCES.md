### 9.2 確認モーダル完了後遷移フロー（全アプリ共通）

**パターン分類**:

#### パターンA: 同画面更新型（最も一般的）
```
[一覧画面] → [アクションボタン] → [確認モーダル] → [確認]
→ [ローディング] → [モーダル閉じる] → [一覧更新] → [成功トースト]
```

適用例:
- 削除操作（ユーザー削除、チケット削除等）
- ステータス変更（承認、却下等）
- 委任設定変更

#### パターンB: 詳細画面遷移型
```
[一覧画面] → [作成ボタン] → [入力モーダル] → [作成]
→ [ローディング] → [モーダル閉じる] → [詳細画面へ遷移] → [成功トースト]
```

適用例:
- 新規作成（チケット作成、提案作成等）
- 複製操作

#### パターンC: 完了画面遷移型
```
[操作画面] → [実行ボタン] → [確認モーダル] → [署名・実行]
→ [Processing画面] → [Complete画面] → [戻るボタン → Dashboard]
```

適用例:
- Lock/Unlock操作
- Challenge提出
- Emergency Pause
