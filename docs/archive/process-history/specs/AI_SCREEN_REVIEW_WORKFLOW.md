# AI Screen Review Workflow

> **Version**: 1.0
> **Date**: 2026-01-25
> **Purpose**: AIが実際の画面を操作してUI/UXの問題を検出するワークフロー

---

## 概要

このドキュメントは、Claude Code + Playwright MCPを使用して、AIが実際のブラウザで画面を操作し、視覚的な問題を検出するワークフローを定義します。

### 解決する問題

- ボタンを押した後の遷移先が未実装
- 文字の見切れ、レイアウト崩れ
- モバイル表示での問題
- 実際の操作フローでの使いにくさ

---

## 前提条件

### 1. Playwright MCP Serverの設定

`.claude/settings.json` に以下が設定されていること:

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-server-playwright"],
      "env": {
        "PLAYWRIGHT_HEADLESS": "true"
      }
    }
  }
}
```

### 2. 開発サーバーの起動

```bash
cd apps/web && pnpm dev
# http://localhost:3000 で起動
```

---

## ワークフロー

### Phase 1: 自動スクリーンショット取得

各画面のスクリーンショットを自動取得してレビュー:

```
Claude指示例:

「Consumer Appの全画面のスクリーンショットを取得して、
以下の観点でレビューしてください:

1. 文字の見切れ、重なり
2. ボタンのタップ領域（44px以上）
3. レイアウトの崩れ
4. モバイル表示での問題

URL一覧は docs/specs/URL_REFERENCE.md を参照」
```

### Phase 2: 遷移フローテスト

ユーザーフローに沿って画面遷移をテスト:

```
Claude指示例:

「田中さん（35歳、暗号資産初心者）として、
以下のユーザーフローを実際に操作してください:

1. http://localhost:3000/ja/consumer/landing を開く
2. 「今すぐ始める」ボタンをクリック
3. オンボーディングを完了
4. ダッシュボードで「ロックする」をクリック
5. ロック完了まで進む

各ステップで:
- スクリーンショットを取得
- 迷った点、分かりにくい点を報告
- 遷移先が正しいか確認」
```

### Phase 3: ペルソナ体験レビュー

ペルソナ視点での体験評価:

```
Claude指示例:

「田中さんとして /ja/consumer/dashboard を操作し、
以下の観点で体験をレビューしてください:

## 理解しやすさ
- 専門用語（Dilithium, STARK等）が説明されているか
- 「量子耐性」の意味が分かるか
- 次に何をすべきか明確か

## 操作性
- ボタンが押しやすいか
- 戻るボタンが機能するか
- エラー時に何が起きたか分かるか

## 安心感
- 資産額が見やすいか
- 操作の確認画面があるか

各項目についてスクリーンショット付きでレポート」
```

## E2Eテストとの連携

AI画面レビューで発見した問題は、E2Eテストとして追加:

```typescript
// レビューで発見: 「すべての履歴を見る」ボタンが遷移しない
test('すべての履歴を見る → history', async ({ page }) => {
  await page.goto('/ja/consumer/dashboard');
  await page.click('a:has-text("すべての履歴を見る")');
  await expect(page).toHaveURL(/\/consumer\/history/);
});
```
