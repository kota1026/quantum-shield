Core Principles

5つの不変原則（変更不可）

1
完全量子耐性 (Quantum Resistance)
NIST準拠の量子耐性アルゴリズムのみ使用。ECDSA, SHA-256は使用禁止。
CP-1
2
Self-Custody
ユーザーが自身の秘密鍵を管理。サーバーに秘密鍵を保存することは禁止。
CP-2
3
Time Lock存在
Time Lockを0にすることは不可。通常24時間、緊急7日間のロック期間。
CP-3
4
Slashing存在
Slashingメカニズムの削除は不可。不正行為への抑止力として機能。
CP-4
5
透明性 (Transparency)
全てオンチェーンで検証可能。オフチェーン秘密計算は禁止。
CP-5

Emergency Procedures

緊急時の対応手順

1
異常を検知したら
まず上長（松本さん）に報告。自己判断でPauseしない。
2
状況を記録
スクリーンショット、ログ、時刻を記録。Slackの#incidentチャンネルに投稿。
3
Pause判断（Admin以上）
Admin権限保持者がPause必要性を判断。Super Admin 2名以上で実行。
4
復旧作業
原因特定 → 修正 → テスト → Security Council承認 → Resume
5
ポストモーテム
インシデントレポートを作成し、コミュニティに公開。
