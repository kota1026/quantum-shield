# Prover戦略会議 議事録

**日時**: 2024年12月21日 15:00 JST
**参加エージェント**: 11名全員
**議題**: Prover問題の解決策検討

---

## 📋 背景

現在のパイプライン:
```
Dilithium署名 → Plonky2 STARK → SP1 zkVM → Groth16 → Ethereum L1
                                            ↑
                                    問題箇所:
                                    1. 重い（証明生成時間）
                                    2. 量子脆弱（bn254 pairing）
```

## 🎯 検討した選択肢

| 選択肢 | 概要 | メリット | デメリット |
|--------|------|----------|------------|
| A. SP1 + Succinct Network | 証明生成を外部委託 | 自前Prover不要、既存コード活用 | 外部依存、Groth16依存継続 |
| B. Plonky3 (Circle STARK) | M31フィールド使用 | 量子耐性、高速 | 書き直し必要 |
| C. Starknet (L2) | L2でSTARK検証 | ガス安い、STARK native | Cairo書き直し |
| D. Aligned Layer | 検証レイヤー外部委託 | 実装簡単 | 新サービス、信頼性未知 |
| E. ハイブリッド | 短期A + 中期B/C | 段階的移行 | 複雑性増加 |

---

## 🛡️ Purpose Guardian 意見

**理念との整合性評価:**
- Groth16依存 = 量子脆弱 = 長期的には理念違反
- ただし「移行計画が明確」であれば短期的には許容
- **推奨**: 選択肢E（ハイブリッド）
