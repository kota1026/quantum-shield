# Part 1: 認証方式マトリックス

| プレイヤー | 認証方式 | 2FA | 生体認証 | セッション時間 |

| Delegate | Wallet (SIWE) | - | オプション | 永続 |
| Proposer | Wallet (SIWE) | - | オプション | 永続 |

| Challenger | Wallet (SIWE) | - | - | 永続 |
| Security Council | Wallet + 2FA | ✅必須 | ✅推奨 | 1時間（操作毎） |
| Purpose Committee | Wallet + 2FA | ✅必須 | ✅推奨 | 1時間 |
| Service Provider | Email + Password | ✅必須 | ✅推奨 | 8時間 |
| QS Staff | Email + Password | ✅必須 | ✅必須 | 4時間 |

## 2.2 SIWE Message構造（EIP-4361）

```
${domain} wants you to sign in with your Ethereum account:
${address}

${statement}

URI: ${uri}
Version: ${version}
Chain ID: ${chainId}
Nonce: ${nonce}
Issued At: ${issuedAt}
Expiration Time: ${expirationTime}
```

### セキュリティポイント

| 要素 | 目的 | 実装 |
|------|------|------|
| Nonce | リプレイ攻撃防止 | サーバー生成、1回限り、5分有効 |
| Domain | フィッシング防止 | 許可ドメインのみ |
| Expiration | セッション制限 | 署名有効期限 |
| Chain ID | チェーン固定 | 正しいチェーンのみ許可 |

# Part 3: Email + Password + 2FA

## 3.1 対象システム

- Enterprise Admin
- QS Admin

## 3.2 パスワード要件

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                          パスワード要件                                          │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  • 最低12文字                                                                   │
│  • 大文字を含む（A-Z）                                                          │
│  • 小文字を含む（a-z）                                                          │
│  • 数字を含む（0-9）                                                            │
│  • 記号を含む（!@#$%^&*）                                                       │
│  • 過去5回のパスワードと異なること                                              │
│  • ユーザー名・メールアドレスを含まない                                         │
│                                                                                 │
│  ハッシュ化: Argon2id                                                           │
│  • memory: 65536 KB                                                             │
│  • iterations: 3                                                                │
│  • parallelism: 4                                                               │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## 3.3 2FA方式

| 方式 | 推奨度 | 説明 |
|------|:------:|------|
| TOTP | ✅ 推奨 | Google Authenticator, Authy, 1Password |
| WebAuthn | ✅ 推奨 | YubiKey, TouchID, FaceID |
| SMS | ⚠️ 非推奨 | SIMスワップリスク |

## 3.5 追加セキュリティ

| 機能 | 実装 |
|------|------|
| ログイン試行制限 | 5回失敗 → 15分ロック |
| 新規デバイス検知 | Email確認必須 |
