# Quantum Shield ユーザージャーニーマップ
## Player Journeys - 体験ベースのUI設計

> **Version**: 1.0  
> **日付**: 2026-01-05  
> **目的**: 各プレイヤーの体験を「ストーリー」として設計し、画面構成を導く  
> **前提**: STEP_D_WHY_QUANTUM_SHIELD.md を読んでいること

---

## このドキュメントの使い方

UI/UXは「機能の配置」ではなく「対話」です。

このドキュメントでは、各プレイヤーが：
- どんな**動機**でサービスに出会い
- どんな**感情**を抱きながら体験し
- どんな**理解**を得て
- どんな**行動**を取るか

を「ストーリー」として描きます。

**各画面は、このストーリーを支えるために存在します。**

## 2.3 各フェーズの詳細

### Phase 1: 認知「なんだこれ？」

| 項目 | 内容 |
|------|------|
| 状況 | Twitter（X）で「量子コンピュータ対応ブリッジ」の投稿を見る |
| 感情 | 好奇心、漠然とした不安 |
| 行動 | リンクをクリックしてLP訪問 |
| 必要画面 | **Landing Page** |
| 伝えること | 量子脅威を簡潔に、QSなら安全という安心感 |

### Phase 2: 理解「なるほど、そういうことか」

| 項目 | 内容 |
|------|------|
| 状況 | 「詳しく知る」をクリック、説明ページを読む |
| 感情 | 「本当に危ないんだ...」→「このサービスなら大丈夫そう」 |
| 疑問 | 本当に安全？操作は難しくない？引き出せる？ |
| 必要画面 | **How It Works**, **Security Explainer**, **FAQ** |
| 伝えること | 図解での仕組み説明、「なぜ24時間か」の理由 |

### Phase 3: 登録「やってみよう」

| 項目 | 内容 |
|------|------|
| 状況 | 「今すぐ始める」をクリック、登録開始 |
| 感情 | 期待と少しの緊張（Dilithium鍵？聞いたことない...） |
| 行動 | Wallet接続→鍵生成→公開鍵登録→バックアップ |
| 必要画面 | **Wallet Connect**, **Key Generation**, **Backup**, **Ready** |
| 伝えること | Dilithium鍵の説明、ブラウザ内完結の安心感、バックアップ重要性 |

### Phase 4: 初回Lock「試してみよう」

| 項目 | 内容 |
|------|------|
| 状況 | まずは少額（0.1 ETH）を試しにロック |
| 感情 | 慎重さ、期待 |
| 行動 | 金額入力→手数料確認→署名→完了 |
| 必要画面 | **Lock Input**, **Lock Confirmation**, **Lock Processing**, **Lock Success** |
| 伝えること | 手数料内訳、「すぐ引き出せない」の再確認、処理状況可視化 |

### Phase 5: Unlock「引き出してみる」

| 項目 | 内容 |
|------|------|
| 状況 | 1週間後、ロックした0.1 ETHを引き出す |
| 感情 | 期待、少しの不安、待機中の焦り |
| 行動 | Unlock申請→Prover検証待ち→24時間待機→実行 |
| 必要画面 | **Unlock Request**, **Unlock Progress**, **Unlock Ready**, **Unlock Execute**, **Complete** |
| 伝えること | ステータスと残り時間、各ステップの説明、通知設定案内 |

### Phase 6: 継続利用「これなら安心」

| 項目 | 内容 |
|------|------|
| 状況 | 初回Unlock成功、本格利用開始（10 ETH） |
| 感情 | 信頼、安心感 |
| 必要画面 | **Dashboard**, **History**, **Settings**, **Key Management** |

### Phase 7: 退会「もう使わない」

| 項目 | 内容 |
|------|------|
| 状況 | 他サービスへ移行、全資産引き出し |
| 感情 | 確認したい（データ残る？戻れる？） |
| 必要画面 | **Pre-Disconnect Checklist**, **Final Unlock**, **Account Disconnect**, **Goodbye** |
| 伝えること | 残資産確認、何が残り何が消えるか、再接続可能 |

# Part 6: UI設計原則

## 6.1 コミュニケーションとしてのUI

| 原則 | 説明 | 例 |
|------|------|-----|
| **「なぜ」を説明** | すべての待機・制限に理由を | ❓アイコンで詳細表示 |
| **不安を先回り** | 懸念を予測して解消 | 「鍵はデバイス内のみ」 |
| **進捗を可視化** | 今どこにいるかを明示 | ステップインジケーター |
| **次を明確に** | 迷わせない導線 | CTAは1つに絞る |
| **価値で語る** | 技術用語を避ける | ❌Dilithium→✅量子耐性の鍵 |

## 6.2 ❓ヘルプパターン

すべての「なぜ？」に答える：

```
┌─────────────────────────────────────────────────────────────┐
│  ❓ なぜ24時間待つのですか？                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  この24時間は「安全期間」です。                             │
│                                                             │
│  万が一、あなたの鍵が盗まれて不正な引き出しが               │
│  行われた場合でも、この期間中に異常を検知し、               │
```
