## 1. veQS 詳細設計

### 1.1 基本仕様（UNIFIED_SPEC.mdより）

| 項目 | 内容 |
|------|------|
| ロック期間 | 1週間〜4年 |
| 重み計算 | 投票力 = QS量 × (残りロック期間 / 最大ロック期間) |
| 最大ブースト | 4倍（4年ロック時） |

### 1.2 追加定義

#### 1.2.1 veQS移転可否

| 項目 | 決定 |
|------|------|
| 移転 | **不可（non-transferable）** |
| 理由 | 証券性リスク軽減、投票権の売買防止 |
| 例外 | Delegation（投票権の委任のみ） |

#### 1.2.2 ロック解除ルール

**採用方式: 早期解除（ペナルティ付き）**

| 解除種類 | 条件 | ペナルティ |
|---------|------|-----------|
| 通常解除 | ロック期間満了後 | なし |
| 早期解除 | いつでも可能 | あり |

**早期解除ペナルティ計算:**
```
ペナルティ = 残りロック期間 / 最大ロック期間 × ロック量 × 50%

例: 4年ロックを2年で解除（100,000 QS）
ペナルティ = 2/4 × 100,000 × 50% = 25,000 QS
返還額 = 100,000 - 25,000 = 75,000 QS
```

**ペナルティの行先:** Treasury

#### 1.2.3 Delegation（投票権委任）

| 項目 | 設定 |
|------|------|
| 委任対象 | 投票権のみ（報酬受領権は委任不可） |
| 委任先 | 任意のアドレス（制限なし） |
| 委任単位 | 全量 or 分割（%指定可） |
| 変更頻度 | いつでも変更可能 |
| 委任チェーン | 不可（再委任禁止） |
| 自己投票 | 委任中も可能（委任先が投票した場合は委任先優先） |

**制限について:**
- 委任先に特別な制限は設けない
- 既存の10%議決権上限ルールが適用される
- Prover兼任禁止はSecurity Councilのみ（既存ルール維持）

## 2. Proposer（提案者）要件

### 2.1 役割

| 項目 | 内容 |
|------|------|
| 目的 | ガバナンス提案の作成・提出 |
| 参入 | Permissionless（閾値あり） |

### 2.2 提案権の閾値

| 提案種類 | 閾値 | QS換算（総供給1B） |
|---------|------|-------------------|
| パラメータ変更 | 0.1% | 1,000,000 QS |
| Treasury支出 | 0.25% | 2,500,000 QS |
| アップグレード | 0.5% | 5,000,000 QS |
| Council変更 | 1% | 10,000,000 QS |

### 2.3 提案プロセス

```
【提案フロー】

1. Forum投稿（オフチェーン）
   ├── 提案内容の詳細説明
   ├── 最低3日間の議論期間
   └── テンプレートに従った記載

2. Temperature Check（オプション）
   ├── Snapshot投票（オフチェーン）
   ├── Quorum: 1%
   └── コミュニティの方向性確認

3. オンチェーン提案提出
   ├── 提案閾値を満たすveQS保有者が提出
   ├── 閾値未満の場合、Delegateに委任依頼可能
   └── 提案データのIPFSハッシュを含む

4. 議論期間（7日）
   ├── 提案内容の変更不可
   └── Forum/Discordで議論継続

5. 投票期間（7日）
   ├── veQS保有者による投票
   └── For / Against / Abstain

6. Time Lock（7日）
   ├── 可決後、実行までの待機期間
   └── Emergency対応はCouncilが短縮可能

7. 実行
   └── 自動実行 or Council実行
```

### 2.4 Quorum要件

| 提案種類 | Quorum | 可決条件 |
|---------|--------|---------|
| パラメータ変更 | 4% | 過半数 |
| Treasury支出 | 4% | 過半数 |
| アップグレード | 8% | 過半数 |
| Council変更 | 15% | 過半数 |

## 3. Delegate（委任先）要件

### 3.1 役割

| 項目 | 内容 |
|------|------|
| 目的 | 委任された投票権を行使 |
| 参入 | Permissionless（誰でもDelegate登録可能） |
| インセンティブ | なし（名誉職）※将来的に報酬検討 |

### 3.2 Delegate登録

| 項目 | 内容 |
|------|------|
| 登録方法 | オンチェーン登録（自己申告） |
| 必須情報 | 名前/ハンドル、ステートメント（ポリシー説明） |
| オプション | SNSリンク、過去の実績、専門分野 |
| 更新 | いつでも可能 |
| 削除 | いつでも可能（委任は自動解除） |

### 3.3 Delegate評価指標

| 指標 | 説明 |
|------|------|
| 委任されたveQS | 総委任量 |
| 自己保有veQS | 自分のveQS |
| 投票参加率 | 過去の投票参加% |
| 投票傾向 | For/Against比率 |
| アクティブ期間 | 登録からの経過日数 |

## 4. Delegator（委任者/Token Holder）要件

### 4.1 役割

| 項目 | 内容 |
|------|------|
| 目的 | 投票権を委任 or 直接投票 |
| 資格 | veQS保有者 |

### 4.2 委任フロー

```
1. Delegate一覧から選択
2. 委任量を指定（全量 or %）
3. トランザクション実行
4. 委任完了（即時反映）
```

### 4.3 委任解除

| 項目 | 内容 |
|------|------|
| 解除方法 | いつでも可能 |
| 反映 | 即時 |
| 進行中投票 | 解除前の投票は有効 |

## 5. Security Council 詳細化

### 5.1 構成

| Phase | 構成 | 合計 |
|-------|------|------|
| Phase 2 | 財団3 + 外部3 | 6名 |
| Phase 3-4 | 財団3 + 外部3 + コミュニティ3 | 9名 |

### 5.2 選出プロセス

#### 財団枠（3名）
| 項目 | 内容 |
|------|------|
| 選出 | QS財団が任命 |
| 変更 | Token Vote必要（15% Quorum） |

#### 外部枠（3名）
| Phase | 選出方法 |
|-------|---------|
| Phase 2 | 財団が外部専門家を招聘 |
| Phase 3+ | Token Voteで選出 |

#### コミュニティ枠（3名）- Phase 3+
| 項目 | 内容 |
|------|------|
| 立候補要件 | veQS 0.1%以上保有 |
| 選出方法 | Token Vote（上位3名当選） |
| 投票期間 | 14日 |
| 任期 | 1年（最大3期） |

### 5.3 権限詳細

| アクション | 必要承認 | Time Lock | 備考 |
|-----------|---------|-----------|------|
| Emergency Pause | 5/9 | なし | 最大72時間 |
| 緊急アップグレード | 7/9 | 48時間 | 事後Token Vote必須 |
| Prover停止 | 5/9 | なし | 即時執行 |
| Slash異議判定 | 7/9 | なし | Challenge結果の異議 |
| パラメータ変更 | Token Vote優先 | 7日 | Council単独不可 |
| Veto | 6/9 | なし | 理念違反の提案のみ |

### 5.4 報酬・制約

| 項目 | 内容 |
|------|------|
| 報酬 | 年間$50K相当の$QS |
| 支払い | 四半期ごと |
| 兼任禁止 | Prover運営不可 |
| 任期 | 1年（再選可、最大3期） |
| 解任 | Token Vote（15% Quorum、過半数） |

### 5.5 運営

| 項目 | 内容 |
|------|------|
| 定例会議 | 月1回（議事録公開） |
| 緊急会議 | 随時（24時間以内に招集） |
| 連絡手段 | 専用Slack/Discord + 緊急電話 |
| 透明性 | 投票記録はオンチェーン公開 |

## 6. Purpose Committee 詳細化

### 6.1 役割

| 項目 | 内容 |
|------|------|
| 目的 | Core Principles（5原則）の保護 |
| 人数 | 3名 |
| 性質 | 「憲法裁判所」的役割 |

### 6.2 Core Principles（保護対象）

| # | 原則 | 説明 |
|---|------|------|
| CP-1 | 完全量子耐性 | NIST準拠の量子耐性アルゴリズムのみ使用 |
| CP-2 | Self-Custody | ユーザーが自身の秘密鍵を管理 |
| CP-3 | Time Lock存在 | Time Lockを0にすることは不可 |
| CP-4 | Slashing存在 | Slashingメカニズムの削除は不可 |
| CP-5 | 透明性 | 全てオンチェーンで検証可能 |

### 6.3 選出プロセス

| Phase | 選出方法 |
|-------|---------|
| Phase 1-2 | 財団任命 |
| Phase 3+ | Token Vote（特別Quorum 20%） |

#### 立候補要件（Phase 3+）
| 項目 | 要件 |
|------|------|
| veQS保有 | 0.5%以上 |
| 専門性 | 暗号学、ブロックチェーン、法律のいずれか |
| 推薦 | Security Council 2名以上の推薦 |

### 6.4 権限

| 権限 | 内容 | 発動条件 |
|------|------|---------|
| 理念チェック | 全提案の理念整合性確認 | 全提案に対して自動 |
| 事前却下 | 理念違反提案を投票前に却下 | 2/3合意 |
| 解釈権 | 曖昧なケースの解釈 | 2/3合意 |
| 異議申立対応 | 却下/解釈への異議はToken Voteで最終判断 | Token Vote 20% Quorum |

### 6.5 報酬・制約

| 項目 | 内容 |
|------|------|
| 報酬 | 年間$30K相当の$QS |
| 支払い | 四半期ごと |
| 任期 | 2年（再選可、最大2期） |
| 兼任禁止 | Security Council兼任不可 |

## 7. 関連会社の10%上限判定

### 7.1 背景

「単一エンティティ10%上限」は定義されているが、関連会社・ペーパーカンパニー対策が必要。

### 7.2 関連会社の定義

以下のいずれかに該当する場合、「関連会社」として合算:

```
1. 議決権の過半数を直接または間接に保有
2. 役員の過半数が共通
3. 実質的支配関係にある（事実認定）
4. 協調行動の合意がある（書面または推定）
```

### 7.3 判定プロセス

| ステップ | 内容 | 担当 |
|---------|------|------|
| 1. 自己申告 | 5%以上保有時に関連会社を開示 | 保有者 |
| 2. モニタリング | オンチェーン分析で集中検知 | QS財団 |
| 3. Challenge | 第三者が関連性をChallenge可能 | 誰でも |
| 4. 判定 | Security Councilが判定 | Council 5/9 |
| 5. 制裁 | 議決権無効化（超過分） | 自動執行 |

### 7.4 制裁

| 違反種類 | 制裁 |
|---------|------|
| 未開示（過失） | 超過分の議決権無効化 |
| 虚偽開示（故意） | 全議決権無効化 + 6ヶ月投票禁止 |
| 協調行動 | 関連者全員の超過分無効化 |
