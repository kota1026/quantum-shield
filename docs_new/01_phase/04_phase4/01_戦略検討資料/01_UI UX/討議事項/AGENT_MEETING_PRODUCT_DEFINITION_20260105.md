## 会議の背景

Phase 4でUI/UXを作成し可視化したことで、以下の課題が顕在化：

1. 各プレイヤーの役割定義が曖昧

## 議題2: プレイヤー役割構想

### 初期認識からの発展

会議中の議論を通じて、当初認識していたプレイヤーに加え、以下の観点が不足していることが判明：

1. **サービス提供構造**: Platform Operator / Service Provider
2. **ガバナンス分散化**: Proposer / Delegate / Delegator 等

### 論点1: Prover登録要件（CSO提起）

**現状の仕様書（SEQUENCES.md #5）:**
- 最低Stake: $400K-$500K
- HSM使用: 必須
- 2-of-3マルチシグ: 必須
- 法的契約: 必須

**未定義事項:**
1. 技術審査の具体的プロセスは？
2. HSM使用の証明方法は？
3. 法的契約の締結相手は？（QS財団？）
4. Phase別の承認フローの詳細は？
5. Proverの地理的分散要件は？

### 論点2: Edition別Prover要件（CBO提起）

| 要件 | Enterprise | Decentralized |
|------|-----------|---------------|
| 参入 | 契約ベース（厳格） | Stake + 自動承認 |
| 審査 | QS財団による面談 | 技術要件のみ |
| 責任 | 契約上の賠償責任 | Slashingのみ |
| 報酬 | 固定 + 変動 | Stake比例 |

### 論点3: Operatorの定義（CTO提起）

| 役割 | 責任 | 例 |
|------|-----|-----|
| **L3 Node Operator** | L3ノードの運営 | 4ノードの運営者 |
| **Prover Operator** | Proverインフラ運営 | HSM + 署名システム |
| **Bridge Operator** | ブリッジ全体の運営 | Enterprise版顧客 |

### 論点4: 用語整理（Legal提起）

| 現行用語 | 提案 | 理由 |
|---------|------|------|
| Guardian | **Observer** | 法的責任を示唆しない |
| 監視ボット | **Monitor** | 技術的役割を明確化 |
| Challenger | Challenger（維持） | 法的に問題なし |

### 論点5: インセンティブ設計（CFO提起）

| プレイヤー | インセンティブ | ペナルティ |
|-----------|--------------|-----------|
| User | 安全な資産移動 | N/A |
| Prover | 手数料報酬（50%配分） | Quadratic Slashing |
| Observer | Whistleblower報酬（$100K） | N/A |
| Challenger | Slash額の60% | Bond没収（敗訴時） |
| L3 Node Operator | インフラ報酬（Treasury） | N/A |
| Security Council | 年間$50K報酬 | 任期制限 |

### 論点6: 攻撃ベクトル分析（Red Team提起）

| プレイヤー | 攻撃ベクトル | 対策状況 |
|-----------|------------|---------|
| Prover | 共謀攻撃 | ⚠️ Quadratic Slashingあるが、5社全員共謀は100%Slashでも足りない可能性 |
| Prover | HSM鍵漏洩 | ⚠️ 2-of-3要件あるが、全鍵漏洩時の対策は？ |
| L3 Node | BFT攻撃（2/4共謀） | ⚠️ 4ノード中2ノード掌握で停止可能 |
| Observer | 偽Alert | ✅ Challenge Bondで抑止 |
| Challenger | 悪意のChallenge | ✅ Bond没収で抑止 |

### 追加論点: サービス提供構造（CEO提起）

**背景**: UI/UX検討を通じて、以下の視点が不足していることが判明：
- サービスを自社サービスとしてエンドユーザーに提供する企業（Service Provider）
- サービスを他のサービサーに提供する管理者的立場（Platform Operator = 我々）

### 追加論点: ガバナンス分散化関連（CEO提起）

分散型運営を想定した場合、以下のプレイヤーの定義が必要：

| プレイヤー | 役割 | 現状 |
|-----------|------|------|
| Proposer | ガバナンス提案を作成・提出 | ⚠️ 誰でも可能？閾値は？ |
| Delegate | 投票権を委任される人 | ⚠️ veQSにDelegation機能あるか？ |
| Delegator | 投票権を委任するToken Holder | ⚠️ 未定義 |
