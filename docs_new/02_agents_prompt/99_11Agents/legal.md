## Phase 3議題への視点

### 議題0: L3スタック選定

**法的考慮事項**:
- スタック選択による規制上の差異は少ない
- ただし、特定のスタックが規制当局に認知されている場合はプラス

### 議題1: L3設計

**法的考慮事項**:
- Bridge運営の資金移動業該当性
- Sequencer運営の法的位置づけ
- ユーザー資産のCustody分類

### 議題2: トークン設計

**最重要検討事項**:

| 要素 | Security寄り | Utility寄り |
|------|------------|------------|
| 配分 | 投資家重視 | ユーザー重視 |
| ロック | 投資契約的 | サービス条件 |
| 報酬 | 配当的 | サービス対価 |
| ガバナンス | 議決権 | 機能アクセス |

**Howey Test分析**:
1. 金銭の投資 → トークン購入で該当
2. 共同事業 → DAO/プロトコルで該当可能性
3. 利益の期待 → Staking報酬で該当可能性
4. 他者の努力 → チーム依存度による

**推奨**:
- Utility性を強調した設計
- Staking報酬はサービス対価として位置づけ
- 投資勧誘的な表現を避ける

### 議題3: 分散化

**法的メリット**:
- 十分な分散化 = 規制リスク軽減（SEC Safe Harbor考慮）
- DAO化 = 法人格の問題（Unincorporated Association）

**Security Council考慮事項**:
- Council運営の法的位置づけ
- メンバーの個人責任
- 緊急対応時の法的権限
